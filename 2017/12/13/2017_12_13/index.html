HEXO无法运行的灾难

发表于 2017-12-13 | 分类于 搭建博客
字数统计: 343 字 | 阅读时长 ≈ 1 分钟

#HEXO突然崩了怎么办【首次安装也可参考】

这是一篇记录性的博文：

起因：在研究NEXT主题中的小功能时，某个包安装失误造成整个hexo崩盘；

背景：在教研室熄灯前半个小时发生，心如乱麻，忙中出错，网上寻遍无果；

表现：运行任何hexo前缀的命令，显示“Local not found hexo in xxx”

解决思路：因为找不到纠错方法，所以采取最笨的——重新安装hexo的方式，也可供首次安装参考；

解决方法：

1.将blog文件夹中的_config.yml文件、theme中新添的主题包、source文件夹复制别处，清空整个blog文件夹；

2.重新安装hexo，依次执行：

  • npm install -g cnpm –registry=https://registry.npm.taobao.org
  • cnpm install hexo-cli -g
  • cnpm link hexo 【此命令非常玄学，是成败关键】
  • hexo -v查看是否安装成功；

3.将hexo初始化：hexo init后将复制的三部分文件替代init产生的对应新文件，然后执行hexo s -g在本地查看效果；

4.重新部署git：cnpm install hexo-deployer-git –save，安装完毕后hexo d进行上传；

【环境变量路径其实我也手动添加了hexo的路径，但是不知道有没有直接关系TAT】

总结：配置需谨慎，不要美化不成功能全崩。
